## Getting Started with the Custom Agent

The fastest way to start building your custom agent is using our **Express Starter Kit**. This repository provides a just a basic express server with a few tools & a basic authentication.

### Repository Overview

**Repository**: [https://github.com/upsurgeio/bhindi-starter-kit-express](https://github.com/upsurgeio/bhindi-starter-kit-express)

The starter contains a complete MongoDB agent with two powerful database tools:

1. **createDocument** - Insert new documents into MongoDB collections
2. **listDocuments** - Query and retrieve documents with filtering, sorting, and pagination

<Note>
  This agent requires a MongoDB connection and supports any MongoDB-compatible
  database including MongoDB Atlas, local instances, and Docker containers.
</Note>

### Environment Configuration

<Warning>
Make sure you have a MongoDB instance running and accessible. The agent requires a valid MongoDB connection URI.
</Warning>

The agent doesn't require environment variables as it accepts the MongoDB URI through request headers, making it flexible for different database connections.

### Testing Your MongoDB Agent

<Steps>
<Step title="Get Available Tools">
First, check what tools are available:

```bash
curl -X GET http://localhost:8080/tools
```

<Check>
You should see both `createDocument` and `listDocuments` tools in the response.
</Check>
</Step>

<Step title="Create a Document">
Insert a new document into your MongoDB collection:

```bash
curl -X POST http://localhost:8080/tools/createDocument \
  -H "Content-Type: application/json" \
  -H "x-mongo-db-uri: mongodb://localhost:27017/testdb" \
  -d '{
    "collection": "users",
    "document": {
      "name": "John Doe",
      "email": "john@example.com",
      "age": 30
    },
    "database": "testdb"
  }'
```

<Check>
The response should include the `insertedId` of your new document.
</Check>
</Step>

<Step title="List Documents">
Retrieve documents from your collection:

```bash
curl -X POST http://localhost:8080/tools/listDocuments \
  -H "Content-Type: application/json" \
  -H "x-mongo-db-uri: mongodb://localhost:27017/testdb" \
  -d '{
    "collection": "users",
    "database": "testdb",
    "filter": {},
    "limit": 10
  }'
```

<Check>
You should see the document you just created in the response.
</Check>
</Step>
</Steps>

### Advanced Usage Examples

<Accordion title="Filtering Documents">
```bash
# Find users older than 25
curl -X POST http://localhost:8080/tools/listDocuments \
  -H "Content-Type: application/json" \
  -H "x-mongo-db-uri: mongodb://localhost:27017/testdb" \
  -d '{
    "collection": "users",
    "database": "testdb",
    "filter": {"age": {"$gte": 25}},
    "limit": 5,
    "sort": {"name": 1}
  }'
```
</Accordion>

<Accordion title="Complex Document Creation">
```bash
# Create a user with nested profile data
curl -X POST http://localhost:8080/tools/createDocument \
  -H "Content-Type: application/json" \
  -H "x-mongo-db-uri: mongodb://localhost:27017/testdb" \
  -d '{
    "collection": "users",
    "document": {
      "name": "Jane Smith",
      "email": "jane@example.com",
      "profile": {
        "bio": "Software Engineer",
        "location": "San Francisco",
        "skills": ["JavaScript", "Python", "MongoDB"]
      },
      "createdAt": "2024-01-15T10:30:00Z"
    }
  }'
```
</Accordion>

### Tool Specifications

| Tool               | Description                                   | Required Parameters      | Optional Parameters                                 |
| ------------------ | --------------------------------------------- | ------------------------ | --------------------------------------------------- |
| **createDocument** | Insert new documents into MongoDB collections | `collection`, `document` | `database` (defaults to "default")                  |
| **listDocuments**  | Query documents with filtering and pagination | `collection`             | `database`, `filter`, `limit`, `sort`, `projection` |

<Tip>
  The agent supports any MongoDB query operators in the `filter` parameter,
  giving you full flexibility for complex queries.
</Tip>
